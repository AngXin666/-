# MuMu 模拟器路径配置说明

## 快速配置（推荐）

### 方法一：选择盘符自动搜索（最简单）

1. 点击"选择盘符"按钮
2. 选择 MuMu 模拟器安装的盘符（如 C:\ 或 D:\）
3. 程序会自动搜索该盘符下的所有 MuMu 安装目录（需要 1-3 分钟）
4. 找到后自动填入路径

**提示**：
- 盘符搜索会扫描整个盘符，可能需要 1-3 分钟
- 如果找到多个安装位置，程序会自动选择第一个
- 你可以在日志中看到所有找到的路径
- 如果未找到，请尝试其他盘符

### 方法二：使用自动检测

1. 点击"自动检测"按钮
2. 程序会自动扫描常见安装位置（C:\ 和 D:\ 的常见路径）
3. 如果找到多个，选择你使用的版本
4. 点击"确定"完成配置

**提示**：
- 自动检测速度快，但只检测常见路径
- 如果 MuMu 安装在非标准位置，请使用"选择盘符"功能

---

## 常见问题

### Q1: 为什么搜索需要 1-3 分钟？

**A:** 程序会递归扫描整个盘符下的所有目录，查找 MuMu 模拟器的安装位置。扫描时间取决于：
- 盘符大小
- 文件数量
- 磁盘速度

### Q2: 找到多个安装位置怎么办？

**A:** 程序会自动选择第一个找到的路径。如果需要使用其他路径：
1. 查看日志中显示的所有路径
2. 重新点击"选择盘符"搜索
3. 或使用"自动检测"功能

### Q3: 搜索失败怎么办？

**A:** 
1. 确认 MuMu 模拟器已安装在该盘符
2. 尝试其他盘符（如果安装在其他盘）
3. 使用"自动检测"功能
4. 联系技术支持

---

## 手动查找路径（高级）

如果自动搜索失败，可以手动查找路径：

### 方法 1：通过快捷方式查找

1. 右键点击桌面上的"MuMu模拟器"图标
2. 选择"打开文件所在位置"
3. 复制地址栏中的路径
4. 联系技术支持提供路径信息

### 方法 2：通过任务管理器查找

1. 启动 MuMu 模拟器
2. 打开任务管理器（Ctrl + Shift + Esc）
3. 找到"MuMu模拟器"相关进程
4. 右键 → 打开文件所在位置
5. 复制路径并联系技术支持

### 常见安装位置

MuMu 模拟器通常安装在以下位置之一：

**MuMu 12.0 版本：**
```
C:\Program Files\Netease\MuMuPlayer-12.0\shell
D:\Program Files\Netease\MuMuPlayer-12.0\shell
```

**MuMu 旧版本：**
```
C:\Program Files\Netease\MuMu\nx_main
D:\Program Files\Netease\MuMu\nx_main
C:\Program Files\Netease\MuMu\nx_device\12.0\shell
D:\Program Files\Netease\MuMu\nx_device\12.0\shell
```

## 路径填写规则

### ✅ 正确的路径示例

以下任何一种路径都可以：

1. **主控制台目录**（推荐）
   ```
   D:\Program Files\Netease\MuMu\nx_main
   ```

2. **设备控制目录**
   ```
   D:\Program Files\Netease\MuMu\nx_device\12.0\shell
   ```

3. **上级目录**（程序会自动查找子目录）
   ```
   D:\Program Files\Netease\MuMu
   D:\Program Files\Netease\MuMu\nx_device
   ```

### ❌ 错误的路径示例

```
D:\Program Files\Netease\MuMu\MuMuPlayer.exe  ❌ 不要包含可执行文件名
C:\Users\用户名\Desktop\MuMu快捷方式.lnk    ❌ 不要使用快捷方式路径
```

## 验证配置是否正确

配置完成后，程序会自动验证：

✅ **成功标志：**
- 显示"✓ 模拟器路径有效"
- 显示"找到 ADB: xxx\adb.exe"
- 可以正常检测运行中的实例

❌ **失败标志：**
- 显示"✗ 模拟器路径无效"
- 显示"未找到 adb.exe"
- 无法检测实例

## 常见问题

### Q1: 提示"找不到模拟器"怎么办？

**A:** 按以下步骤排查：
1. 确认 MuMu 模拟器已安装
2. 使用"自动检测"功能
3. 如果自动检测失败，手动查找路径（参考上面的方法）
4. 确保路径中包含 `adb.exe` 文件

### Q2: 我的 MuMu 安装在非标准位置怎么办？

**A:** 
1. 使用"浏览"按钮选择安装盘符
2. 或者手动输入完整路径
3. 程序会自动在该路径下搜索 ADB

### Q3: 配置后还是无法使用怎么办？

**A:** 
1. 检查 MuMu 模拟器版本（建议使用 12.0 或更高版本）
2. 确认至少有一个模拟器实例在运行
3. 重启程序后重新配置
4. 联系技术支持

---

## 技术说明

程序会自动在以下位置查找 `adb.exe`：
- 配置路径本身
- 配置路径的 `shell` 子目录
- 配置路径的上级目录
- 配置路径的 `nx_device/12.0/shell` 子目录
- 递归搜索（最多 3 层）

所以即使路径不完全准确，程序也能自动找到 ADB。
